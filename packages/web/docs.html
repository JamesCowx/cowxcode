CowxCode Docs

Everything you need to run, configure and extend the CowxCode coding agent.

Getting started

CowxCode ships as a Windows desktop app, but the engine is plain JavaScript you can run anywhere Node runs. 

git clone https://github.com/your-org/cowxcode
cd cowxcode
npm install
npm run build
npm run start:desktop

On first launch, open Settings from the sidebar, choose a provider, paste your API key, and pick a project folder. Then start chatting.

How it works

CowxCode drives a tight agent loop: your message is sent to a provider, the model streams a response, and when it emits a tool call (as JSON in a fenced block) the engine executes it against your project and feeds the result back to the model. This repeats until the task is complete. 

user → provider → assistant text
                 → tool call (```json)
tool runs on disk → result → provider → … → done

Providers

CowxCode is provider-agnostic. Configure one in Settings (stored locally in your user data folder as cowxcode.json).

  • OpenAI (compatible) — any OpenAI-style endpoint.
  • OpenCode Go — $10/month subscription at opencode.ai/zen/go/v1.
  • OpenCode Zen — pay-as-you-go at opencode.ai/zen/v1.
  • Anthropic — Claude models via the Messages API.
  • Google Gemini — Google's Gemini models.
  • Mistral — Mistral Large, Small, Codestral and more.
  • Groq — fast open-weight models (Llama, Mixtral, Gemma).
  • xAI Grok — Grok 2/3 models.
  • Perplexity — Sonar online-search models.
  • OpenRouter — unified gateway to 100+ models.
  • DeepSeek — direct DeepSeek API.
  • Ollama — local models at http://localhost:11434/v1.
  • LM Studio — local models at http://localhost:1234/v1.

Config file

The on-disk config shape is simple:

{
  "provider": {
    "provider": "openai",
    "baseUrl": "https://api.openai.com/v1",
    "apiKey": "sk-…",
    "model": "gpt-4o"
  }
}

Swap provider for anthropic, ollama or lmstudio as needed. Keys never leave your machine except to reach the provider you configured.

Tools

The agent can act on your project through these built-in tools. Each is exposed in the Tools tab of the desktop app.

  • read_file — read a text file.
  • write_file — create or overwrite a file.
  • list_dir — list a directory.
  • run_command — execute a shell command.
  • search_files — regex search across files.

Sessions

Each chat is a session with full message history. Use New Session in the header to reset context. The engine keeps a system prompt plus your conversation turns.

Architecture

The monorepo is split into focused packages:

  • packages/core — provider clients, tools, agent loop.
  • packages/desktop — Electron app with native Windows UI.
  • packages/web — this documentation site.
